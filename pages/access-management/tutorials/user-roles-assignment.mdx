# User Roles Assignment

To assign a user for roles, you can use the `assignRolesToUser()`. This method need `userId` and `roleKeys`.

<Callout type="info">
Please refer to the API documentation for the [`assignRolesToUser()`](https://jupiter-pp.roq-platform.com/docs/#mutation-assignRolesToUser) method.
</Callout>

The `roles()` method can be used to query any available roles. For example, to get all available roles as a **Super Admin**:

```js
const rolesAvailable = await roqClient.asSuperAdmin().roles({
	limit: 10,
	withUserCount: true
})
```

Let say that we want to assign a role **Marketing** to the user with id `7877d2d0-dea7-473e-a158-57eca3123906` then you should use **Marketing** `key` role from the output above, which is `marketing`:


```js
const assignStatus = await roqClient.asSuperAdmin().assignRolesToUser({
	userId: "7877d2d0-dea7-473e-a158-57eca3123906",
	roleKeys: ["marketing"]
})
```
